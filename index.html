Principles

The idea is that you have a central machine acting as a centralized storage, that once a day (this is configurable of course) connect with one or more endpoints in order to make backup of them.

It's based upon pre-existing open source softwares

  • swap_horiz SSH

    Standard remote tool for operating between systems
    home page

  • swap_vert RSNAPSHOT

    filesystem snapshot utility based on rsync
    home page

  • code Python

    Well known open source language

Easy Backup consists of two (Debian) packages, one for the storage system and one for the endpoint, that have to be installed in the right machines.

Other than saving the files, are available some tools to help the administrator to restore the backups, maintaining the right permissions, creating tar archive ready to be used in the destination system.
